Fase 1 — Criação da Pasta do Prospect

Navegar até Operações → Prospects.
✓

✓

Fase 2 — Estrutura de Fechamento
Execute quando o lead avançar para fechamento.

01_Contrato
✓
Criar 01_Contrato dentro da pasta da empresa.
✓
Dentro de 01_Contrato, criar 01_Assinado.
✓
Quando o lead assinar, inserir o PDF do contrato assinado dentro de 01_Assinado.
💡 O contrato elaborado não precisa ser salvo — fica no Bitrix. Salvar aqui apenas o PDF assinado pela empresa.
02_Documentações Base → 01_Societário
✓
Criar 02_Documentações Base e dentro criar 01_Societário.
✓
Inserir o Contrato Social renomeado como:
CONTRATO SOCIAL - [NOME DA EMPRESA]
  • Ex: CONTRATO SOCIAL - LUMAVA ENGENHARIA LTDA — em CAIXA ALTA
✓
Para cada alteração contratual renomear como:
ALTERAÇÃO CONTRATUAL - [NOME DA EMPRESA]
  • Se houver mais de uma: ALTERAÇÃO CONTRATUAL 1, ALTERAÇÃO CONTRATUAL 2…
02_Documentações Base → 02_Responsável Legal
▶ Se houver 1 responsável legal
✓
Criar 02_Responsável Legal dentro de 02_Documentações Base.
✓
Inserir a CNH ou Identidade renomeada como:
[Nome Completo do Responsável]
  • Ex: CARLOS ALBERTO FERREIRA
✓
Inserir o Comprovante de Residência renomeado como:
Comprovante de Endereço - [Nome Completo]
  • Ex: Comprovante de Endereço - Carlos Alberto Ferreira
  • Deve estar atualizado
✓
Se casado, inserir a Certidão de Casamento renomeada como:
Certidão de Casamento - [Nome Completo]
▶ Se houver 2 ou mais responsáveis legais
✓
Criar 02_Responsável Legal dentro de 02_Documentações Base.
✓
Criar uma subpasta com o nome completo de cada responsável dentro de 02_Responsável Legal:
  • Ex: 📁 Carlos Alberto Ferreira
  • Ex: 📁 Mariana Costa Souza
✓
Dentro de cada pasta individual, inserir e renomear os documentos de cada responsável:
  • CNH/Identidade → Nome Completo do Responsável
  • Comprovante de Residência → Comprovante de Endereço - Nome Completo
  • Certidão de Casamento → Certidão de Casamento - Nome Completo (se casado)
💡 Repetir para todos os responsáveis legais. Cada um com sua própria pasta.
02_Documentações Base → 03_Certificado Digital
✓
Criar 03_Certificado Digital dentro de 02_Documentações Base.
✓
Inserir o certificado e arquivo de texto com a senha nomeado como:
[Nome da Empresa] - Senha [senha]
  • Ex: Lumava Engenharia - Senha 123456
🚨 Acesso restrito. Certifique-se de que apenas pessoas autorizadas têm acesso a esta pasta.

Fase 3 — Migração para Clientes
Execute somente após assinatura e pagamento da entrada confirmados.
🔒 Só execute esta fase após confirmação de assinatura e pagamento da entrada. Migrar antes é um erro de processo.
📁 Operações
├── 📁 Prospects  ← mover daqui
└── 📁 Clientes
    └── 📁 L  (exemplo: LUMAVA → pasta L)
        └── 📁 [Nome da Empresa]  ← para aqui
Migração
✓
Confirmar que o lead assinou o contrato e pagou a entrada.
✓
Navegar até Operações → Clientes.
✓
Identificar a letra correta pela inicial da Razão Social:
  • Ex: LUMAVA ENGENHARIA → pasta L
  • Ex: TECH SOLUÇÕES → pasta T
  • Usar sempre a inicial da primeira palavra da Razão Social
✓
Mover a pasta de Prospects para dentro da letra correta em Clientes.
Verificação final
✓
A pasta não está mais em Prospects — foi movida corretamente.
✓
A pasta está dentro da letra correta em Clientes.
✓
O contrato assinado em PDF está salvo em 01_Contrato → 01_Assinado.
✓
Toda a estrutura da Fase 2 está completa e corretamente nomeada.
🏛️ Cliente oficialmente integrado. Pasta em Clientes, organizada e pronta para a equipe operacional.
